MODEL CITIZENS

The criteria for what constitutes a good citizen vary from time to time and from culture to culture. Although everyone has a different point of view, in most cases the social system operates basically according to the "model citizen" (this is very subjective). And the word also constrains people's moral boundaries.
As people move into the Digital Age, applications are constantly promoted on the network, and the popularity scope also increases accordingly. In order to regulate users' behaviour, the operating parties make agreements according to the laws in real society to restrict them to become a "model citizen".
However, "What should we do? Why do we need to follow the rules? What happens if I violate it?" questions like these will still be on our minds. We may not be able to judge good or evil, but we need to understand the consequences of our actions and understand how they affect us.

Introduction


My initial response to the prompt was to think about How to let the user identify the credit of the other party. Most of us are slightly wary of strangers when interacting with others because we can't fully disengage from those outsides of ourselves. But that level of vigilance diminishes with time and familiarity, and that's when accidents can happen. There are other issues to consider, such as how to raise user awareness. In other words, users need to be made aware of the serious consequences of a breach of trust, so that they understand the importance of normative behaviour.

My aims for this project include:
 Enhancing the integrity awareness of network users and let them understand the meaning of their actions.
 Changing people's attitudes so they can trust each other more clearly
 Simulating the virtual scene and the appearance of the characters to show what to do in different situations

Process


      1. Wrote the base code

      First, I learned the basics of how to build a web page through the tutorial videos I found on the website. Then, depending on my project, I needed to create a number of subsites to convert, so I created multiple profiles and coded the corresponding elements. For me who didn't know anything about programming, building the site was pretty hard, so it took me longer than I thought it would.

      2. Web layout typesetting

      After writing the basic code, I used CSS to help me beautify my page, layout and typesetting. However, limited by the lack of relevant knowledge, only the simplest and most crude way to design the website UI.
      3. Revaluate the project

      At first, My idea was to transplant the AR function on the mobile phone to the computer and generate the user's personal profile with other personal information. However, many operations and techniques are difficult to achieve with my ability. There are many other planning points like this. So I reassessed my project and tried to simplify it as much as possible.
      4. Changes

      After rethinking my plan, my project was mostly done in HTML, CSS, Java, and similar programming syntax. First, because there is no cloud database to store user data, each web page can only be designed, but there is no user profile. Then, instead of relying on A 3D scene to complete the presentation, the case was changed to a question-and-answer page.
      5. Comment

      Because there is no database, the account cannot be connected for communication. Therefore, a simple web chat simulator is used to conduct conversation simulations to render the desired web site.
